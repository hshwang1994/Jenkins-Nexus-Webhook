# Nexus 백업/복구 절차

## 백업 대상

Nexus 복구에는 **PostgreSQL DB와 blob store를 반드시 함께 백업**해야 한다.
DB만 백업하면 메타데이터는 있지만 파일이 없는 상태가 되고,
blob만 백업하면 파일은 있지만 Nexus가 인식하지 못한다.

| 대상 | 경로 | 용도 |
|------|------|------|
| PostgreSQL DB | `pg_dump nexusdb` | asset 메타데이터, repository 설정, 사용자, task 등 |
| Blob store | `/nexus-data/blobs/default/` | 실제 파일 바이너리 |
| Nexus 설정 | `/nexus-data/etc/` | nexus.properties, nexus-store.properties |

## 백업 절차

### PostgreSQL

```bash
# cron: 매일 02:00 (cloviradmin 또는 root)
sudo -u postgres pg_dump nexusdb | gzip > /var/lib/pgsql/backups/nexusdb_$(date +%Y%m%d).sql.gz
# 7일 보관
find /var/lib/pgsql/backups -name "nexusdb_*.sql.gz" -mtime +7 -delete
```

현재 178/179/180에 daily cron 설치 완료.

### Blob store

```bash
# blob store는 파일 복사 또는 rsync
rsync -avz /nexus-data/blobs/default/ /backup/nexus-blobs/
```

대용량 blob이 있으면 시간이 오래 걸린다. 운영 부하가 낮은 시간(03시)에 실행 권장.

## 복구 절차

1. Nexus 중지: `systemctl stop nexus`
2. PostgreSQL 복원:
   ```bash
   sudo -u postgres dropdb nexusdb
   sudo -u postgres createdb nexusdb -O nexus
   zcat /var/lib/pgsql/backups/nexusdb_YYYYMMDD.sql.gz | sudo -u postgres psql nexusdb
   ```
3. Blob store 복원:
   ```bash
   rsync -avz /backup/nexus-blobs/ /nexus-data/blobs/default/
   chown -R nexus:nexus /nexus-data/blobs/
   ```
4. Nexus 시작: `systemctl start nexus`
5. 복구 후 검증:
   - Admin UI 로그인
   - Browse에서 repository/asset 확인
   - REST API status 200 확인
6. DB와 blob이 불일치하면:
   - Settings > Tasks > Create task > "Repair - Reconcile component database from blob store"
   - 해당 repository 선택 후 Run

## 복구 리허설

연 1회 이상 복구 테스트 권장. 별도 VM에 Nexus를 설치하고 백업 파일로 복원하여 정상 동작을 확인한다.
