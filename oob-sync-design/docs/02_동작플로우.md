# 동작 플로우

## 1. 전체 흐름 요약

```
[포털]
  └─ 파일 업로드/삭제
       ▼
[이천 운영 Nexus]
  ├──(Webhook POST)──▶ [운영 Master]
  └──(Webhook POST)──▶ [개발 Master]
                          │
                    각 Master 내부:
                    리전(ich/chj/yi) 별로
                      1. 후보 조회 → online 필터 → 제외
                      2. 대표 OOB 에이전트 선정
                      3. Ansible playbook 실행
```

---

## 2. 업로드 이벤트 (실시간)

```
[Nexus asset 생성]
  └─ Webhook 발사 (HMAC 서명 포함)
       ├──▶ [운영 Master webhook_dispatch]
       └──▶ [개발 Master webhook_dispatch]
                 │
           HMAC 검증 → master_realtime_sync 트리거
                 │
           리전별:
             1. nodesByLabel("ich && redfish") 후보 조회
             2. online 필터 + exclude_node_names 제거
             3. 대표 OOB 에이전트 선정 (파일보유 → 최근성공 → 이름정렬)
             4. 대표 노드에서 ansible-playbook realtime_sync.yml 실행
                 │
                 ▼
           [대표 OOB 에이전트]
             1. /sync/tmp/<repo>/<asset>.part 에 다운로드
             2. checksum / size 검증
             3. /sync/files/<repo>/<asset> 로 원자적 mv
             4. peer 노드에 rsync over SSH 전파
                 │
                 ▼
           [대상 OOB 에이전트]
             1. /sync/incoming/<repo>/<asset>.part 로 수신
             2. checksum 검증
             3. /sync/files/<repo>/<asset> 로 원자적 mv
```

### Webhook 수신 상세

**수신 플러그인**: Generic Webhook Trigger Plugin

**payload 파싱** (`genericVariables`):

| Nexus JSON path | Jenkins 변수 | 설명 |
|---|---|---|
| `$.id` | `event_id` | 이벤트 식별자 |
| `$.action` | `operation_type` | CREATED → upload, DELETED → delete |
| `$.repositoryName` | `repository_name` | Nexus repository |
| `$.asset.name` | `asset_path` | repository 내 상대 경로 |
| `$.asset.downloadUrl` | `download_url` | 다운로드 URL |
| `$.asset.checksum.sha256` | `checksum_value` | SHA-256 checksum (우선). 없으면 sha1 fallback. |
| `$.asset.size` | `file_size_bytes` | 바이트 |

`X-Nexus-Webhook-Signature` 헤더 → `webhook_signature` 변수. Raw body → `webhook_raw_body` 변수.

### HMAC 서명 검증 절차

1. Jenkins Credentials `nexus-webhook-hmac-secret` 에서 secret key 로드
2. raw body 전체를 HMAC-SHA256 으로 서명하여 hex digest 계산
3. `X-Nexus-Webhook-Signature` 헤더 값과 case-insensitive 비교
4. 불일치 시 한글 5단 로그 + 즉시 실패
5. 일치 시 `master_realtime_sync` Job 을 `wait: false` 로 트리거

### 동시 실행 방지

`master_realtime_sync` 에 `disableConcurrentBuilds()` 적용.

## 3. 삭제 이벤트 (실시간)

```
[Nexus 삭제]
  └─ Webhook (operation_type=delete)
       ▼
[각 Master master_realtime_sync]
  └─ Ansible: quarantine_delete.yml
       ├─ 대표 OOB: /sync/files/... → /sync/quarantine/...
       └─ 대상 OOB: SSH 로 동일하게 quarantine 이동

[전체 점검 시]
  └─ Nexus 기준 재확인 → quarantine 파일 최종 삭제
```

---

## 4. 주기 점검

### 트리거

| 모드 | cron | 범위 |
|---|---|---|
| partial | `H/30 * * * *` | 최근 변경분 + 최근 실패분 + 신규 OOB catch-up |
| full | `H 3 * * *` | 전체 asset 비교 + quarantine 최종 삭제 |

### 처리 절차

1. **Nexus asset 목록 조회** — Python 스크립트(`list_nexus_assets.py`)로 페이지네이션 조회. 부분 점검 시 `lastModified` timestamp 를 기준으로 최근 변경분만 클라이언트 측에서 후처리 필터링한다.
2. **로컬 파일 목록 수집** — `find /sync/files`
3. **최근 실패분 로드** — `/var/lib/oob-sync/recent_failures.json`
4. **누락 = (Nexus - 로컬) + 최근 실패분** → 대표 OOB 에이전트에 다운로드
5. **과잉 = 로컬 - Nexus** → quarantine 이동
6. **quarantine 최종 삭제** (전체 점검에서만)
7. **대상 OOB 에이전트 전파** — 대표 OOB 에이전트가 rsync 로 각 peer 에 전파
8. **실패 기록 초기화**

### 부분/전체 비교

| 항목 | 부분 점검 | 전체 점검 |
|---|---|---|
| Nexus 조회 범위 | 최근 변경분 | 전체 asset |
| 최근 실패분 재시도 | O | O |
| 신규 OOB catch-up | O | O |
| quarantine 최종 삭제 | X | O |
| 보관 | 30개 / 14일 | 14개 / 30일 |

### catch-up 원리

- 주체는 **reconcile** 이다.
- reconcile 이 Nexus 기대 상태 기준으로 대표 OOB 의 `/sync/files` 를 완성한다.
- 그 후 대표 OOB 가 rsync 로 신규 노드에 전파한다.
- `rsync --delete` 는 정리 동작이며 catch-up 의 본질이 아니다.

## 5. 신규 OOB Agent 자동 편입

```
[운영자]
  └─ 신규 OOB Agent 등록 (라벨: ich, redfish 부착)
       ▼
[다음 실시간 동기화 또는 주기 점검]
  └─ nodesByLabel 결과에 신규 노드 자동 포함
       ├─ 실시간: 새 파일이 자동으로 신규 노드에도 fan-out
       └─ 점검: Nexus 기대 상태 기준 누락 파일 식별 → rsync 전파
```

## 6. 상태 점검 (15분)

```
[cron H/15 * * * *]
  └─ master_heartbeat
       └─ 모든 OOB Agent 에 대해:
            1. Jenkins online 여부
            2. /sync 경로 존재
            3. 여유 디스크
            4. 최근 동기화 상태
       └─ 이상 발견 시 한글 로그 출력
```
