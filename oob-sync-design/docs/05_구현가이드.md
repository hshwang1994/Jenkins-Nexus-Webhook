### 확정된 설계 기본값

아래 항목은 설계 기본값으로 확정되었다. 구현 시 별도 확인 없이 적용한다.

|---|---|

## 2. 구현 순서

### Phase 0. 사전 준비 (전제: C1 확정)

1. 운영/개발 Master Jenkins 버전 확인 (LTS 권장)
2. 필수 플러그인 설치
   - Pipeline, Pipeline Utility Steps, Credentials Binding
   - **Generic Webhook Trigger Plugin**
   - SSH Agent, Ansible
3. Jenkins Agent 사전 요건 확인
   - `ansible-playbook`, `rsync`, `ssh`, `sha256sum`, `sha1sum`, `df`, `python3` 사용 가능
4. SSH fan-out 에 사용할 OS 계정 확인 (기본값: `cloviradmin`, 대체 예: `infraops`)
   - 환경에 이미 할당된 계정을 재사용한다. 전용 계정 신규 생성은 필수가 아니다.
   - 지정 계정이 모든 OOB Agent 에 존재하고 SSH 키 인증이 가능해야 한다.
   - 계정 변경 시 반드시 아래 3곳을 함께 맞춘다.
     1. `sync_config.yml` 의 `fanout.ssh_user` 값
     2. Jenkins Credentials `oob-sync-ssh-key` 의 username
     3. 모든 OOB Agent 의 `authorized_keys` 배치 경로(`/home/<계정>/.ssh/authorized_keys`) 및 `/sync`, `/var/lib/oob-sync`, `/opt/oob-sync` 소유권 대상
5. `/sync/{files,tmp,incoming,quarantine}` 디렉터리 생성, 위 OS 계정 소유
6. `/var/lib/oob-sync/` 디렉터리 생성

### Phase 1. 설정/자격 증명 등록

1. `sync_config.yml` 배포 (모든 OOB Agent + Master)
2. Jenkins Credentials 등록 (운영/개발 Master 에 동일 ID)
   - `nexus-webhook-hmac-secret` (secret text)
   - `nexus-oob-sync-readonly` (username/password)
   - `oob-sync-ssh-key` (SSH user private key, username=`fanout.ssh_user` 값. 기본값: `cloviradmin`)
3. Nexus 측
   - 기존 Nexus 계정이 read + browse 권한을 가지고 있으면 재사용 가능. 없으면 전용 read-only 계정 생성.
   - Repository Webhook Capability: 운영 Master 용 1개 + 개발 Master 용 1개 등록, HMAC secret, CREATED/DELETED 만 활성화

### Phase 2. Ansible 산출물 배포

1. `ansible/` 디렉터리를 GitOps repository 에 커밋
2. Jenkinsfile 의 playbook 경로를 실제 `${WORKSPACE}` 경로에 맞춤
3. 단위 검증: OOB Agent 에서 수동으로 `realtime_sync.yml` 실행

### Phase 3. Jenkins Job 등록

1. 운영/개발 Master 에 4개 Job 등록
   - `oob-sync/webhook_dispatch`
   - `oob-sync/master_realtime_sync`
   - `oob-sync/master_reconcile`
   - `oob-sync/master_heartbeat`
2. Generic Webhook Trigger 설정 확인: `token`, `printContributedVariables: false`
3. 빌드 보관 정책 반영 확인

### Phase 4. 기능 검증

1. TC-01 ~ TC-04 (기본 흐름 + 대표 선정 규칙)
2. TC-05, TC-07 (실패 복구, 누락 복구)
3. TC-06 (신규 노드 catch-up)
4. TC-09, TC-11 (Credentials 인증, HMAC 위조 거부)
5. TC-08, TC-12 (quarantine, 대용량)

### Phase 5. 운영 안정화

1. 한글 로그 운영 적합성 검증
2. heartbeat → 모니터링 대시보드 연결
3. 보관 정책 실제 동작 확인
4. 알림 채널 연동 (본 설계 미포함, 운영 환경에 맞춰 추가)

## 4. Jenkins 파라미터 정의

### 4.1. Webhook 수신 구조

**플러그인**: Generic Webhook Trigger Plugin

**endpoint**: `POST http://<jenkins-master>/generic-webhook-trigger/invoke?token=oob-sync-nexus-webhook`

**payload 파싱** (`genericVariables`):

| Nexus JSON path | Jenkins 변수 | 설명 |
|---|---|---|
| `$.id` | `event_id` | 이벤트 식별자 |
| `$.action` | `operation_type` | CREATED → upload, DELETED → delete |
| `$.repositoryName` | `repository_name` | repository |
| `$.asset.name` | `asset_path` | 상대 경로 |
| `$.asset.downloadUrl` | `download_url` | 다운로드 URL |
| `$.asset.checksum.sha256` | `checksum_value` | SHA-256 checksum (우선). sha256 미제공 시 sha1 fallback. |
| `$.asset.size` | `file_size_bytes` | 바이트 |

`X-Nexus-Webhook-Signature` 헤더 → `webhook_signature`, Raw body → `webhook_raw_body`.

### 4.2. HMAC 검증

1. `nexus-webhook-hmac-secret` 에서 key 로드
2. HMAC-SHA256(key, raw body) → hex digest
3. `webhook_signature` 와 case-insensitive 비교
4. 불일치 → 한글 5단 로그 + 실패
5. 일치 → `master_realtime_sync` 트리거

### 4.3. Job 별 파라미터

| Job | 파라미터 |
|---|---|
| webhook_dispatch | Generic Webhook Trigger 자동 주입 |
| master_realtime_sync | event_id, operation_type, repository_name, asset_path, download_url, checksum_algorithm, checksum_value, file_size_bytes |
| master_reconcile | mode (partial / full) |
| master_heartbeat | 없음 (cron) |

## 5. Jenkins Credentials

| ID | 타입 | 용도 | 권한 |
|---|---|---|---|
| `nexus-webhook-hmac-secret` | secret text | Webhook HMAC 검증 | — |
| `nexus-oob-sync-readonly` | username/password | Nexus 다운로드 + REST 조회 | read + browse (기존 계정 재사용 가능) |
| `oob-sync-ssh-key` | SSH user private key | OOB Agent 간 rsync over SSH | `fanout.ssh_user` (기본값: `cloviradmin`) |

### Jenkinsfile 에서 사용

```groovy
withCredentials([
  string(credentialsId: 'nexus-webhook-hmac-secret', variable: 'HMAC_SECRET'),
  usernamePassword(credentialsId: 'nexus-oob-sync-readonly',
                   usernameVariable: 'NEXUS_USER', passwordVariable: 'NEXUS_PASS'),
  sshUserPrivateKey(credentialsId: 'oob-sync-ssh-key',
                    keyFileVariable: 'OOB_SSH_KEY', usernameVariable: 'OOB_SSH_USER')
]) {
  // Jenkins 가 콘솔 로그에서 비밀번호를 자동 마스킹(****)
}
```

### Ansible 전달

Ansible playbook 은 `lookup('env','NEXUS_USER')` 으로 환경변수를 읽는다.
