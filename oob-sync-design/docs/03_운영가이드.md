# 운영 가이드 — 장애 처리 및 로그 규칙

---

## 1. 장애 처리 시나리오

각 시나리오는 **발생상황 / 원인 / 문제 / 조치방법 / 자동 복구 가능 여부** 를 포함한다.

---

### 1.1. 운영 Master 성공, 개발 Master 실패

**발생상황**: Nexus 가 Webhook 을 양쪽에 보냈으나 개발 Master 동기화만 실패.

**원인**: 개발망 OOB Agent 오프라인, Nexus 계정 권한 누락, SSH 키 만료.

**조치방법**:
1. 개발 Master 의 빌드 로그에서 실패 단계(다운로드/검증/전파) 식별.
2. credential / SSH 키 / agent 상태 복구.
3. 다음 부분 점검(30분)이 자동 catch-up.

**자동 복구**: 예.

---

### 1.2. 특정 리전 후보군 전체 오프라인

**발생상황**: 해당 리전 OOB Agent 전부 오프라인.

**조치방법**:
1. Jenkins 노드 관리에서 해당 리전 OOB 에이전트 상태 확인.
2. 오프라인 원인 해소 후 다음 점검에서 자동 catch-up.

**자동 복구**: 예 (복구 후).

---

### 1.3. 대표 OOB 에이전트 Nexus 다운로드 실패

**발생상황**: 대표 OOB 가 3회 재시도 후에도 다운로드 실패.

**조치방법**:
1. 대표 OOB 에서 `curl` 로 download_url 직접 접속.
2. 인증 실패면 `nexus-oob-sync-readonly` credentials 점검.
3. 디스크 부족이면 quarantine/tmp 정리.

**자동 복구**: 예.

---

### 1.4. checksum 불일치

**발생상황**: 다운로드 후 size 또는 checksum 이 Nexus 메타와 다르다.

**조치방법**:
1. 임시 파일은 자동 폐기됨.
2. 다음 점검에서 자동 재다운로드.
3. 반복 실패 시 Nexus asset 메타 확인.

**자동 복구**: 부분적 (재시도 자동, Nexus 메타 자체 오류면 수동).

---

### 1.5. 대상 OOB 에이전트 일부 실패

**발생상황**: 대표 OOB 는 성공했지만 일부 peer 전송/검증 실패.

**조치방법**:
1. 로그에서 실패 노드 확인.
2. 디스크/SSH/네트워크 점검.
3. 다음 점검이 실패 노드만 재전송.

**자동 복구**: 예.

---

### 1.6. 삭제 이벤트 — 대상 파일이 이미 없음

**발생상황**: delete Webhook 왔지만 파일이 이미 없다.

**조치방법**: 별도 조치 없음. 정보 로그만 출력.

---

### 1.7. 신규 OOB Agent catch-up

**발생상황**: 신규 OOB Agent 등록, `/sync/files` 가 비어 있음.

**조치방법**: 별도 작업 없음. 다음 주기 점검(reconcile)이 Nexus 기대 상태 기준으로 누락 파일을 식별하고, 대표 OOB 에이전트가 rsync 로 신규 노드에 전파.

**자동 복구**: 예.

---

### 1.8. Jenkins Credentials 조회 실패

**발생상황**: `withCredentials` 에서 credential ID 조회 실패.

**조치방법**:
1. Jenkins → Credentials 에서 ID/scope 확인.
2. 폴더 권한 필요 시 폴더 credentials 복제.

**자동 복구**: 아니오.

---

### 1.9. Nexus 인증 실패 (401/403)

**조치방법**:
1. `nexus-oob-sync-readonly` credentials 갱신.
2. Nexus 측 role 에 read 권한 확인.

**자동 복구**: 아니오.

---

### 1.10. 빌드 보관 정책 미설정

**조치방법**: 모든 Jenkinsfile 의 `properties([...])` 에 `buildDiscarder(logRotator(...))` 명시. 본 산출물에 이미 포함됨.

**자동 복구**: 아니오 (코드 수정 필요).

---

## 2. 운영 로그 규칙

### 2.1. 작성 원칙

- 분석자가 설계를 모르는 상태에서도 이해 가능한 한국어
- 5단 구조: **제목 / 발생상황 / 원인 / 문제 / 조치방법**
- 내부 코드명은 풀어 쓴다

### 2.2. 상태 로그 필수 키

| 키 | 의미 |
|---|---|
| `event_id` | 이벤트 식별자 |
| `location_code` | 리전 코드 |
| `candidate_oob_nodes` | 라벨 후보 전체 |
| `online_oob_nodes` | online 필터 결과 |
| `offline_oob_nodes` | offline 노드 |
| `excluded_oob_nodes` | exclude 에 의해 제외된 노드 |
| `selected_seed_node` | 대표 OOB 에이전트 |
| `selected_peer_nodes` | 대상 OOB 에이전트 목록 |
| `selection_reason` | 대표 선정 사유 (한글) |

## 3. 한글 로그 템플릿

### T01 — 대표 OOB 에이전트 선정 불가

```
[운영 OOB 파일 동기화 실패]

발생상황:
{{ location_code }} 리전의 OOB 파일 동기화를 시작했지만, 사용 가능한 대표 OOB 에이전트를 찾지 못했습니다.

원인:
{{ location_code }} 리전 OOB 에이전트가 모두 오프라인 상태입니다.

문제:
{{ location_code }} OOB망으로 Nexus 파일을 가져올 수 없습니다.

조치방법:
1. Jenkins 노드 관리 화면에서 해당 리전 OOB 에이전트 상태를 확인합니다.
2. 오프라인 원인을 해소합니다.
3. 에이전트 복구 후 다음 부분 점검이 자동 catch-up 합니다.
```

### T02 — Nexus 다운로드 실패

```
[대표 OOB 에이전트 다운로드 실패]

발생상황:
대표 OOB 에이전트({{ selected_seed_node }}) 가 Nexus 에서 파일을 다운로드하지 못했습니다.

원인:
Nexus 접속 불가, 인증 실패, 네트워크 단절, 디스크 부족 중 하나입니다.

문제:
해당 리전 OOB Agent 들이 최신 파일을 받을 수 없습니다.

조치방법:
1. Credentials(nexus-oob-sync-readonly) 유효성 확인.
2. 대표 OOB 에서 curl 로 download_url 직접 접근.
3. 해결 후 주기 점검이 자동 복구.
```

### T03 — checksum 불일치

```
[checksum 불일치]

발생상황:
{{ checksum_algorithm }} checksum 이 기대값과 다릅니다.
expected = {{ checksum_value }}
actual   = {{ actual_checksum }}

원인:
다운로드 또는 전송 중 데이터 손상, Nexus 메타 불일치.

문제:
손상된 파일 반영 방지를 위해 즉시 중단.

조치방법:
1. 임시 파일은 자동 폐기됨.
2. 다음 부분 점검이 자동 재다운로드.
3. 반복 시 Nexus asset 메타 확인.
```

### T04 — 대상 OOB 에이전트 일부 전파 실패

```
[대상 OOB 에이전트 일부 전파 실패]

발생상황:
일부 대상 OOB 에이전트로의 전송 또는 검증이 실패.
전송실패: {{ fanout_fail_peers }}
검증실패: {{ finalize_fail_peers }}

원인:
네트워크 불안정, 디스크 부족, SSH 키 문제.

문제:
실패 노드만 최신 파일 미반영.

조치방법:
1. 해당 노드 디스크/네트워크/SSH 점검.
2. 다음 부분 점검이 실패 노드만 재전송.
```

### T05 — 디스크 부족

```
[OOB Agent 디스크 부족]

발생상황:
{{ inventory_hostname }} 의 여유 디스크가 기준보다 적습니다.
현재 = {{ free_gb }} GB, 기준 = {{ required_gb }} GB

원인:
quarantine 누적, 대용량 파일 누적, 임시 파일 미정리.

조치방법:
1. quarantine 경로의 오래된 파일 정리.
2. 용량 확보 후 자동 정상화.
```

### T06 — Webhook 서명 검증 실패

```
[Webhook 서명 검증 실패]

발생상황:
Nexus Webhook 서명이 일치하지 않습니다.

원인:
HMAC secret 불일치 또는 위조된 요청.

조치방법:
1. Nexus Capability 의 secret 과 Jenkins Credentials(nexus-webhook-hmac-secret) 일치 확인.
2. 일치 후 Nexus 에서 재전송 또는 수동 트리거.
```

### T07 — Jenkins Credentials 조회 실패

```
[Jenkins Credentials 조회 실패]

발생상황:
파이프라인이 Credentials 에서 자격 증명을 가져오지 못했습니다.

원인:
Credential ID 오기재, credential 삭제, 폴더 권한 문제.

조치방법:
1. Manage Jenkins → Credentials 에서 ID/scope 확인.
2. 필요 시 폴더 credentials 복제.
```

### T08 — 신규 OOB Agent catch-up

```
[신규 OOB Agent catch-up]

발생상황:
{{ location_code }} 리전에 새 OOB Agent({{ new_node_name }}) 가 추가되었습니다.

원인:
운영 확장으로 신규 OOB Agent 등록.

문제:
신규 노드의 /sync/files 가 비어 있어 일시적으로 정합성 부족.

조치방법:
별도 작업 불필요.
다음 주기 점검(reconcile)이 Nexus 기대 상태 기준으로 누락 파일을 식별하고,
대표 OOB 에이전트가 rsync 로 신규 노드에 기존 파일을 전파합니다.
```
