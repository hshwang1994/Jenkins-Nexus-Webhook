# 운영 하드닝 체크리스트

## 1. SSH 보안

### StrictHostKeyChecking

현재: `StrictHostKeyChecking=no` (테스트 편의)

운영 전환:
```bash
# 1. Master에서 모든 Agent의 host key 수집
for IP in 186 189 172 173 174 177; do
  ssh-keyscan -H 10.100.64.$IP >> /var/lib/jenkins/.ssh/known_hosts
done
chown jenkins:jenkins /var/lib/jenkins/.ssh/known_hosts

# 2. Agent에서 same-region peer의 host key 수집
# (ich 186↔189, chj 172↔173, yi 174↔177)
ssh-keyscan -H 10.100.64.189 >> /home/jenkins/.ssh/known_hosts
ssh-keyscan -H 10.100.64.186 >> /home/jenkins/.ssh/known_hosts
# ... 각 지역별 반복

# 3. Jenkinsfile/sync_config.yml에서 StrictHostKeyChecking=no 제거
```

### SSH Key 관리

| 키 | 위치 | 용도 |
|----|------|------|
| Master jenkins key | `/var/lib/jenkins/.ssh/id_rsa` | Master → Agent SCP |
| Agent jenkins key | `/home/jenkins/.ssh/id_rsa` | seed → peer rsync |
| `jenkins-ssh-key` credential | Jenkins Credentials | Jenkinsfile에서 참조 |

운영 전:
- Master/Agent SSH key는 자동 생성됨 (4096-bit RSA)
- private key 권한: `600` (jenkins:jenkins)
- `authorized_keys` 권한: `600`

## 5. 중복 이벤트 방어

- `disableConcurrentBuilds()` 적용 (master_realtime_sync)
- 동일 event_id가 재전송되어도 idempotent (파일 덮어쓰기)
- Delete 이벤트: quarantine 이동은 idempotent (이미 없으면 skip)

## 6. 대용량 파일

- Master staging 경로: `/var/lib/oob-sync/staging`
- staging 디스크 여유 기준: `min_free_disk_bytes` (50 GB)
- 전송 후 staging 파일 자동 삭제 (post.always)
- rsync `--partial --inplace` 옵션으로 전송 중단 시 재개 가능

## 7. 로그 정책

- buildDiscarder: realtime 50건/14일, reconcile 30건/14일, full 14건/30일, heartbeat 30건/7일
- 한글 5단 로그 구조: 발생상황/원인/문제/조치방법
- credentials masking: withCredentials 블록 사용
