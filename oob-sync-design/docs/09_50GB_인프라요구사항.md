# 50GB 운영 목표 인프라 요구사항

## 현재 상태

| 서버 | 현재 디스크 | 현재 여유 | 역할 |
|------|----------|----------|------|
| 178 Nexus | 61GB | 30GB (blob 정리 후) | Primary Nexus |
| 198 Master | 61GB | 46GB | 운영 Jenkins |
| 199 Master | 61GB | 46GB | 개발 Jenkins |
| Agent 12대 | 61GB | 47-51GB | OOB Agent |

## 50GB 목표 필수 요구사항

| # | 항목 | 현재 | 필요 | 이유 | 우선순위 |
|---|------|------|------|------|---------|
| 1 | **178 Nexus blob store** | 61GB (/ 파티션) | **500GB+ 별도 LV** | 50GB 원본 + soft-delete 잔존(50GB) + 여유(100GB+) | **필수** |
| 2 | **198/199 Master staging** | 61GB | **200GB+** | curl 다운로드(50GB) + staging 임시(50GB) + 여유 | **필수** |
| 3 | **Agent /sync** | 61GB | **200GB+** | files(50GB) + incoming(50GB) + quarantine + 여유 | **필수** |
| 4 | **blob store 별도 마운트** | / 파티션 공유 | **별도 LV 또는 NFS** | Nexus DB/log와 blob 분리. OS 디스크 포화 방지 | **필수** |
| 5 | **rsync timeout** | 3600초 | **7200초** | 50GB 단일 리전 전송 = ~30분(1Gbps), 여유 포함 | 권장 |
| 6 | **blob 정리 운영 절차** | 없음 | **주기적 cleanup + 모니터링** | 삭제 후 디스크 미회수 방지 | **필수** |

## 산정 근거

### 178 Nexus
- 50GB 파일 업로드 시: 원본(50GB) + 임시(50GB) + blob soft-delete 잔존 = **최소 150GB**
- 안전 여유 포함: **500GB 권장**
- OS(20GB) + DB(1GB) + log(2GB) + blob(500GB) = 별도 마운트 필수

### 198/199 Master
- staging: curl 다운로드 50GB + 검증 중 중복 점유 = **100GB**
- OS + Jenkins workspace + logs = **50GB**
- 합계: **200GB 권장**

### Agent /sync
- /sync/files: 50GB (최대 파일 1개 기준)
- /sync/incoming: 50GB (rsync 수신 중)
- /sync/quarantine: 50GB (삭제 대기)
- 합계: **200GB 권장**
