## 6. Proxy Cache 동작 설명

CE 에디션은 Pro 전용 Content Replication(자동 복제)을 지원하지 않는다. 대신 **Proxy Repository**를 사용한 on-demand cache 방식으로 동일한 목적을 달성한다.

| 구분 | Pro (Content Replication) | CE (Proxy Cache) |
|------|--------------------------|-------------------|
| 복제 방식 | Push (자동, 실시간) | Pull (on-demand, 최초 요청 시) |
| 최초 다운로드 지연 | 없음 (이미 복제됨) | 있음 (Primary에서 캐싱) |
| 이후 다운로드 | 로컬 | 로컬 (캐시) |
| 네트워크 의존성 | 복제 완료 후 독립 | 최초 요청 시 Primary 연결 필요 |
| 비용 | Pro 라이선스 필요 | CE 무료 |

**운영 영향**: OOB Agent가 지역 Secondary에서 파일을 처음 요청할 때만 Primary 네트워크 접근이 필요하다. 이후 동일 파일은 로컬 캐시에서 즉시 응답한다.

## 11. oob-sync-design 연동 준비 정보

# ============================================================
# 계열2: infra-automation-raw (포털 업로드, OOB 동기화)

# → 179/180에는 이 repository의 proxy 없음

# Webhook (infra-automation-raw-hosted → Jenkins)

JENKINS_PROD=http://10.100.64.198:8080
JENKINS_DEV=http://10.100.64.199:8080

# 서비스 계정
NEXUS_SVC_USER=svc_app_install_read

### Jenkins → OOB Agent 배포 구조

```
Source of Truth: 178 infra-automation-raw-hosted
Jenkins는 오케스트레이터 및 중간 전달 허브. 대체 파일 소스가 아님.

Jenkins Master (198/199)
  ├─ Webhook 수신 (178 infra-automation-raw-hosted → component 이벤트)
  ├─ 이천 대표 OOB Agent 1대 → 178에서 직접 다운로드 → 이천 나머지 fan-out
  ├─ 용인 대표 OOB Agent 1대 → 178에서 직접 다운로드 → 용인 나머지 fan-out
  └─ 청주 대표 OOB Agent 1대 → 178에서 직접 다운로드 → 청주 나머지 fan-out
```
