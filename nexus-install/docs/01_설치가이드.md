# Nexus Repository CE 3.91.0 오프라인 설치 가이드

## 1. 개요

에어갭(인터넷 차단) RHEL 9.6 서버에 Sonatype Nexus Repository CE 3.91.0을 오프라인 설치한다. **3-서버 토폴로지**(Primary 1대 + Secondary 2대)를 구성하며, `install.env`의 `NEXUS_ROLE` 값에 따라 설치 스크립트가 자동으로 역할별 동작을 분기한다.

### 아키텍처: Primary + Secondary (Proxy Cache)

```
178 이천 (Primary, raw-hosted)
  ├─ 파일 업로드 수신
  ├─ Webhook -> Jenkins Master
  │
  ├── Proxy Cache ──> 179 용인 (Secondary, raw-proxy)
  └── Proxy Cache ──> 180 청주 (Secondary, raw-proxy)
```

CE 에디션은 Pro 전용 Content Replication을 지원하지 않는다. 대신 **Proxy Repository**를 사용한 on-demand cache 방식으로 동일한 목적을 달성한다. Secondary에 파일 요청이 오면, 로컬 캐시에 없는 경우에만 Primary에서 다운로드하여 캐싱한다.

### 대상 서버

| 역할 | IP | OS | 리소스 | NEXUS_ROLE |
|------|----|----|--------|-----------|
| **Primary** (이천) | 10.100.64.178 | RHEL 9.6 | 4 vCPU / 8 GB | `primary` |
| **Secondary** (용인) | 10.100.64.179 | RHEL 9.6 | 8 vCPU / 16 GB | `secondary` |
| **Secondary** (청주) | 10.100.64.180 | RHEL 9.6 | 8 vCPU / 16 GB | `secondary` |

### 기술 스택

| 구성 요소 | 버전 | 비고 |
|----------|------|------|
| Nexus Repository CE | 3.91.0-07 | linux-x86_64 |
| PostgreSQL | 16.13 | PGDG RPM |
| Nginx | 1.20.1-24.el9 | RHEL 9 기본 |
| Java | Eclipse Temurin 21.0.9+10 | Nexus 번들 내장, INSTALL4J_JAVA_HOME_OVERRIDE 강제 |
| OS | RHEL 9.6 x86_64 | |

## 2. 사전 요구 사항

- RHEL 9.6 최소 설치 완료
- root 또는 sudo 권한
- **Primary (178)**: 최소 4 vCPU, 8 GB RAM, 50 GB 디스크
- **Secondary (179, 180)**: 권장 8 vCPU, 16 GB RAM, 50 GB 디스크
- 번들 준비용 인터넷 연결 가능 머신 1대
- **Secondary 설치 전 Primary가 먼저 가동 중이어야 한다** (Proxy Repository가 Primary URL을 참조)

---

## 3. NEXUS_ROLE 설명

`install.env`의 `NEXUS_ROLE` 변수가 전체 설치/설정 동작을 결정한다.

| NEXUS_ROLE | 동작 | Repository | Webhook |
|-----------|------|-----------|---------|
| `primary` | Raw Hosted Repository 생성 | `application-install-raw-hosted` | 운영/개발 Jenkins로 등록 |
| `secondary` | Raw Proxy Repository 생성 (Primary를 원격 URL로 지정) | `application-install-raw-proxy` | 등록하지 않음 |

**install.sh**는 역할에 무관하게 동일하게 실행된다 (OS 패키지, PostgreSQL, Nexus, Nginx 설치). **configure.sh**에서 NEXUS_ROLE에 따라 Repository 유형과 Webhook 등록이 분기된다.

### JVM 설정 분기

| 항목 | primary (8 GB) | secondary (16 GB) |
|------|---------------|-------------------|
| Heap | -Xms2g -Xmx2g | -Xms4g -Xmx4g |
| Direct Memory | -XX:MaxDirectMemorySize=2g | -XX:MaxDirectMemorySize=4g |
| PG shared_buffers | 512MB | 1024MB |

## 4. 설치 순서

### Phase 1: 번들 준비 (인터넷 연결 머신)

```bash
# 1. 스크립트 전송
scp -r nexus-install/ cloviradmin@10.100.64.178:/opt/nexus-bundle-src/

# 2. 번들 준비 실행
ssh cloviradmin@10.100.64.178
sudo bash /opt/nexus-bundle-src/scripts/prepare_online_bundle.sh
```

이 스크립트가 수집하는 항목:
- Nexus 3.91.0-07 Linux x86_64 아카이브 + SHA256 체크섬
- PostgreSQL 16 RPM (서버, contrib, libs) + 의존성
- Nginx RPM + 의존성
- 시스템 유틸리티 RPM

### Phase 2: 오프라인 설치 (각 서버)

**중요: 178 Primary를 먼저 설치하고, 이후 179/180 Secondary를 설치한다.**

```bash
# 1. install.env에서 NEXUS_ROLE 설정
#    178: NEXUS_ROLE=primary
#    179, 180: NEXUS_ROLE=secondary
vi /opt/nexus-bundle/config/install.env

# 2. 설치 실행 (3대 모두 동일한 스크립트)
sudo bash /opt/nexus-bundle/scripts/install.sh
```

설치 순서 (9단계, 역할 무관):
1. 시스템 유틸리티 RPM + OpenSSL 설치
2. PostgreSQL 16 설치 + 초기화 + DB/사용자 생성 + pg_trgm (RAM에 따라 최적화)
3. Nexus OS 사용자 생성
4. Nexus 아카이브 추출 + **번들 JDK 21 강제** (INSTALL4J_JAVA_HOME_OVERRIDE) + 설정 배포
5. systemd 서비스 등록
6. Nginx 설치 + SELinux httpd_can_network_connect + 리버스 프록시
7. Nexus 시작 + 부팅 대기 (최대 5분)
8. 방화벽 포트 개방 (80, 8081)
9. **JDK 21 런타임 검증** (PID -> Java 경로 -> 버전 확인)

### Phase 3: 초기 설정 (REST API, NEXUS_ROLE 분기)

```bash
sudo bash /opt/nexus-bundle/scripts/configure.sh
```

#### Primary (NEXUS_ROLE=primary) 설정 항목:
1. EULA 수락
2. admin 비밀번호 변경 -> `Goodmit0802!`
3. 익명 접근 비활성화
4. Blob Store 확인 (`default`)
5. Raw **Hosted** Repository 생성 (`application-install-raw-hosted`)
6. 읽기 전용 서비스 계정 생성 (`svc_app_install_read`)
7. Webhook Capability 자동 등록 (운영/개발 Jenkins)

#### Secondary (NEXUS_ROLE=secondary) 설정 항목:
1. EULA 수락
2. admin 비밀번호 변경 -> `Goodmit0802!`
3. 익명 접근 비활성화
4. Blob Store 확인 (`default`)
5. Raw **Proxy** Repository 생성 (`application-install-raw-proxy`, 원격 URL: 178 Primary)
6. 읽기 전용 서비스 계정 생성 (`svc_app_install_read`)
7. Webhook -- **건너뜀** (Secondary에서는 미등록)

### Phase 4: 검증

```bash
bash /opt/nexus-bundle/scripts/validate_nexus_install.sh
```

Primary 검증 항목 (15개, 26 체크포인트):
1. 서비스 상태 (PostgreSQL, Nexus, Nginx)
2. 포트 리스닝 (5432, 8081, 80)
3. Nexus REST API (status + writable)
4. admin 인증
5. Repository 존재 (`application-install-raw-hosted`)
6. 서비스 계정 인증
7. 파일 업/다운로드 + 익명 거부
8. PostgreSQL (연결 + pg_trgm + 버전)
9. Nginx 프록시
10. 디스크 여유
11. **JDK 검증** (번들 JDK 경로 + 버전 21 확인)
12. **/nexus-data 소유권** (nexus 사용자)
13. **Webhook Capability** 등록 확인
14. **Nginx 버전**
15. **SELinux httpd_can_network_connect**

Secondary 검증은 Repository가 `application-install-raw-proxy`이고, Webhook 항목이 WARN(미등록, 정상)으로 표시된다.

## 6. 롤백

```bash

sudo bash /opt/nexus-bundle/scripts/rollback_nexus_install.sh
```

## 7. 트러블슈팅

| 증상 | 원인 | 조치 |
|------|------|------|
| Nexus 5분 내 미기동 | JVM 메모리 부족 | `nexus.vmoptions` 에서 힙 사이즈 조정 (8GB: 2g, 16GB: 4g) |
| PostgreSQL 연결 실패 | pg_hba.conf 인증 | md5 인증 설정 확인 |
| pg_trgm 설치 실패 | contrib 패키지 누락 또는 OpenSSL 버전 부족 | OpenSSL 업데이트 후 contrib 재설치 |
| 포트 80 충돌 | nginx.conf 기본 server 블록 | 기본 server 블록 주석 처리 |
| 502 Bad Gateway | SELinux httpd_can_network_connect=off | `setsebool -P httpd_can_network_connect 1` |
| 403 Forbidden (업/다운로드) | EULA 미수락 | REST API EULA 수락 (configure.sh Step 0) |
| admin.password 없음 | 이미 초기 설정 완료 | install.env 의 비밀번호로 로그인 |
| 번들 체크섬 불일치 | 전송 중 파일 손상 | 번들 재전송 |
| Proxy 다운로드 실패 | Primary 미가동 또는 네트워크 차단 | 178 Primary 서비스 상태 및 방화벽 확인 |
| Proxy 캐시 갱신 안됨 | Nexus 캐시 TTL | Admin -> Repository -> Proxy 설정에서 캐시 만료 시간 조정 |

## 8. Webhook 설정 (Primary 전용)

Webhook은 Primary의 `configure.sh`에서 REST API로 자동 등록된다. Secondary에서는 등록하지 않는다.

### 등록 방법

1. `install.env`에 Jenkins Master URL 설정:
   ```
   WEBHOOK_URL_PROD="http://10.100.64.198:8080/generic-webhook-trigger/invoke?token=infra-automation-nexus-webhook"
   WEBHOOK_URL_DEV="http://10.100.64.199:8080/generic-webhook-trigger/invoke?token=infra-automation-nexus-webhook"
   ```
2. configure.sh 실행 (NEXUS_ROLE=primary일 때만 등록):
   ```bash
   sudo bash /opt/nexus-bundle/scripts/configure.sh
   ```

URL이 빈 값이면 해당 Webhook 등록을 건너뛴다.

### 등록되는 Webhook 사양

| 항목 | 값 |
|------|-----|
| API | POST /service/rest/v1/capabilities |
| type | webhook.repository |
| Repository | **`infra-automation-raw-hosted`** (계열2, 포털 업로드 대상) |
| Event Types | component (생성/삭제) |
| Secret Key | install.env의 WEBHOOK_HMAC_SECRET |
| Token | `infra-automation-nexus-webhook` |

운영 Master용 1개, 개발 Master용 1개 총 2개 자동 등록된다.
